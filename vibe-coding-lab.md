# Vibe Coding Lab

This lab is written for end users who want to create a **profile website** from a rough idea. You do not need a perfect brief, code terms, or a separate prompt for every move. Start with the person, the vibe, and the goal, then let GitHub Copilot shape that into a practical profile website draft.

## Lab Goal

Turn a rough idea into **1 polished profile website direction**, including:

- **1 clarified website brief**
- **1 recommended one-page structure**
- **1 first website draft**
- **1 refined final direction**
- **1 clear next-step recommendation**

> **Quick Start:** If you want to begin right now, jump to the [Start Once, Then Keep Going](#start-once-then-keep-going) section, copy the prompt, and paste it into Copilot Agent chat. Come back to the rest of this guide when you need help.

## Before You Start

1. Open this project in VS Code.
2. Open GitHub Copilot Chat.
3. Select **Agent** mode.
4. Keep the same chat thread throughout the lab.
5. If Copilot needs more context, describe these things in plain language:
   - whose profile website this is
   - who the website is for
   - what impression the site should create
   - what visitors should do next
   - what sections you already know you want

> Tip: You do not need to explain technology choices. In this lab, the goal is to get to a strong profile website concept quickly.

## How the Lab Should Flow

### 1. Turn the Vibe into a Website Brief

In this step, Copilot will translate your rough idea into a clean website brief before writing the page.

Signs it went well:

- the person and purpose are obvious
- the audience is clear
- the desired impression is easy to understand

### 2. Define the One-Page Structure

In this step, Copilot will propose the best section order for a profile website.

Signs it went well:

- the section order supports trust and clarity
- the page feels focused instead of overloaded
- the CTA appears in the right places

### 3. Create the First Website Draft

In this step, Copilot will create the first usable draft of the profile website.

Signs it went well:

- the headline is strong and specific
- the page explains who the person is and why they matter
- the content is concrete enough to review immediately

### 4. Refine the Website with Plain-Language Feedback

In this step, Copilot will improve the draft based on short feedback such as "make it more premium" or "make the personal brand clearer".

Signs it went well:

- the feedback creates a visible improvement
- the website becomes clearer, not busier
- the page still feels like one coherent profile

### 5. Package the Final Direction

In this step, Copilot will summarize the final direction, explain what changed, and tell you where files were saved.

Signs it went well:

- the final website direction is easy to share with someone else
- file names and locations are clear if files were created
- the next step is practical, such as refining copy or adding real projects

## Step-by-Step Prompt Examples

Use these prompts one by one if you want a guided workflow.

> **Note:** Anything in `[brackets]` is a placeholder. Replace it with your own content. If you are using the default scenario, use the values from the **Default Lab Scenario** table above.

### Step 1 — Clarify the profile website brief

```text
Help me create a one-page profile website from a rough idea.
The website is for [name or role].
It should feel [tone or vibe].
The audience is [target audience].
The goal is [main goal].
Turn this into a clear website brief before creating anything.
```

### Step 2 — Define the page structure

```text
Based on that brief, propose the best one-page structure for the profile website.
Tell me what sections to include, what order they should appear in, and why.
Keep it focused and practical.
```

### Step 3 — Create the first draft

```text
Now create the first draft of the profile website.
Include a strong hero section, a short about section, expertise or skills, selected work or proof, and a contact section.
Use copy that feels modern, confident, and premium.
Make it clear, modern, and easy to scan.
```

### Step 4 — Refine the direction

```text
Keep the same structure, but refine the website.
Make it feel more [premium / warm / credible / creative].
Reduce anything that feels generic or too wordy.
Strengthen the headline, CTA, and proof of expertise.
```

### Step 5 — Finalize and summarize

```text
Summarize the final profile website direction.
Explain the section structure, the overall tone, the strongest message, and the best next step.
If you created files, tell me exactly where they were saved.
```

## Copy-Ready Feedback Prompts

If you want a faster iteration loop, copy one of these:

- `make the profile feel more premium and less generic`
- `make the headline clearer and more memorable`
- `reduce the text and improve scanability`
- `make the site feel more personal without sounding casual`
- `show me 2 tighter hero variations for the same profile`
- `make the proof of expertise stronger and easier to trust`
- `keep the same direction, but make it more polished and modern`

## What a Best Prompt Should Include

A strong prompt for this lab usually has **8 parts**.

1. **Who the website is about**  
   Example: founder, designer, consultant, student, creator

2. **Who the audience is**  
   Example: clients, recruiters, partners, employers, collaborators

3. **What the website must achieve**  
   Example: build trust, get inquiries, show expertise, win interviews

4. **Tone or vibe**  
   Example: modern, premium, warm, confident, minimalist, creative

5. **Required sections**  
   Example: Hero, About, Skills, Projects, Testimonials, Contact

6. **Main CTA**  
   Example: Book a Call, View Work, Contact Me, Download Resume

7. **Constraints**  
   Example: one-page only, easy to scan, not too wordy, professional but human

8. **Expected workflow**  
   Example: first clarify the brief, then define structure, then create the draft, then refine it

## Practical Tips

- Start with person, audience, goal, and tone before talking about visuals.
- Ask for one strong version before asking for many alternatives.
- Use short refinement prompts instead of rewriting the whole brief.
- If you do not have real content yet, ask Copilot to use editable placeholder content.
- Keep the whole workflow in one chat thread.
- Ask Copilot to confirm exactly what it created and where it saved the files before you finish.
